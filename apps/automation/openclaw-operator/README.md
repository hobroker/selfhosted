# `openclaw-operator`

> Kubernetes operator for self-hosting OpenClaw AI agents with production-grade security, observability, and lifecycle management

Source Code: https://github.com/openclaw-rocks/openclaw-operator
Chart: `oci://ghcr.io/openclaw-rocks/charts/openclaw-operator`

## Installing/upgrading

```sh
# Register / update the Application resource
kubectl apply -f application.yaml

# Then sync the workload - via ArgoCD UI or:
argocd app sync openclaw-operator
```

### Manual Helm (without ArgoCD)

```sh
kubectl apply -f config
helm install openclaw-operator \
  oci://ghcr.io/openclaw-rocks/charts/openclaw-operator \
  --namespace openclaw-operator-system \
  --create-namespace \
  -f values.yaml
```

## Instance

The `OpenClawInstance` CR (`config/openclaw-instance.yaml`) deploys a managed OpenClaw instance named `kira` in the `openclaw` namespace. The operator reconciles it into a StatefulSet, Service, NetworkPolicy, RBAC, PVC, and Ingress.

```sh
kubectl get openclawinstances -n openclaw
# NAME   PHASE     AGE
# kira   Running   2m

kubectl get pods -n openclaw
# NAME     READY   STATUS    AGE
# kira-0   1/1     Running   2m
```

### Connecting to the Control UI

```sh
kubectl port-forward -n openclaw svc/kira 18789:18789
# Open http://localhost:18789#token=<gateway-token>
```

Retrieve the auto-generated gateway token:

```sh
kubectl get secret kira-gateway-token -n openclaw -o jsonpath='{.data.token}' | base64 -d
```

## Secrets

Environment variables are sourced from `infisical-openclaw-secret` (synced via the InfisicalSecret CR in `config/`):

| Name                | Description                      |
| ------------------- | -------------------------------- |
| `ANTHROPIC_API_KEY` | Anthropic API key (`sk-ant-...`) |

> **Note:** `OPENCLAW_GATEWAY_TOKEN` is auto-generated by the operator — no need to provision it manually.

## Storage

| PVC                    | Namespace | Size  | Description                                  |
| ---------------------- | --------- | ----- | -------------------------------------------- |
| `data-kira-0`          | `openclaw`| 10Gi  | Agent workspace: config, sessions, skills    |

PVCs are retained on instance deletion (`orphan: true` default). To reattach after recreation, set `spec.storage.persistence.existingClaim: data-kira-0`.

## Observability

Prometheus metrics are scraped via the `ServiceMonitor` created by the operator. Dashboards:

- **OpenClaw Operator** — fleet overview, reconciliation metrics, auto-update panels
- **OpenClaw Instance** — per-instance CPU, memory, storage, network, and pod health
